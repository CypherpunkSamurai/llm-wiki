# Research

<info>
Research grows the wiki, not just the current answer. Follow the main topic and useful side topics, but turn each source into raw material and lasting topic pages so knowledge builds over time.
</info>

<critical>
1. Always save raw references before changing topic pages.
2. Prefer add-only extension of existing pages. Fix old claims only when newer evidence changes the knowledge state or shows the old claim was wrong.
3. Side topics with lasting value become their own raw and topic entries, not chat notes.
4. Give credit when newer findings challenge older claims.
5. Keep archives saved; research updates topics and index, not old snapshots.
</critical>

<tool-choices>
1. Start with `./wiki/index*.md`; use targeted reads before wide searching.
2. When the wiki already covers the topic, read only the relevant pages and the sources needed to fill clear gaps.
3. Reuse the Add pipeline rules for raw capture, topic templates, and binary handling.
</tool-choices>

<step>
1. Search `./wiki/index*.md` for the topic and close related terms.
2. If relevant pages exist, read them and find missing coverage, old claims, weak summaries, or unresolved conflicts.
3. Research only the missing parts. For each new source, save a raw reference first, then add it to the most relevant topic page.
4. If no relevant page exists, research the topic, save raw references, and build one or more new topic pages from them.
5. When research produces side but lasting knowledge, create separate topic entries and cross-link them with `See Also`.
6. Add to existing pages instead of rewriting them from scratch. Fix only when a claim is replaced or wrong, and give clear credit for conflicts.
7. Update `./wiki/index.md` for every new or changed topic page.
8. Add a log entry:
   `## [YYYY-MM-DD HH:MM] research | <primary topic>`
   `- Added: <new article>`
   `- Updated: <updated article>`
   `- Side topic: <side article>`
</step>
